# Handoff: 로딩 상태 스켈레톤 (Loading States)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.LoadingStates`
> 이 패키지는 **전 페이지 공용 로딩 스켈레톤 명세**입니다. Design.EmptyStates와 한 쌍 — 로딩→(스켈레톤)→데이터 0건→(빈 상태) 순으로 이어집니다.

## 기본 블록 (Tailwind 공용 컴포넌트 1개로)
- 시머: `linear-gradient(90deg, #e9edf4 25%, #f4f6fa 50%, #e9edf4 75%)`, `background-size:200% 100%`, **1.6s 무한 루프 좌→우** (`@keyframes sh { 0%{background-position:200% 0} 100%{background-position:-200% 0} }`)
- 프리미티브 4종: 텍스트 바(radius 6, 높이 8~15px) · 아바타/엠블럼 원 · 썸네일/커버(radius 10~14, **한 톤 진한 #e2e7f0**) · 칩(radius 99)
- 폭은 **고정 배열** (55%/80%/40% 등) — 랜덤 금지, 재방문 시 흔들림 방지

## 동작 규칙 (필수)
1. **200ms 지연 후 표시** — 빠른 응답에서 스켈레톤 플리커 방지 (200ms 안에 오면 아예 안 보임)
2. **실제 레이아웃과 동일한 골격** — 로드 완료 시 위치 점프 0px. 카드 크기·아바타 위치·행 높이를 실물과 맞출 것
3. 3초 초과 시 스켈레톤 하단에 "불러오는 중이에요…" 문구 추가
4. **스피너는 버튼 내 제출 동작에만** (로그인, 승인, 저장). 페이지·목록 로딩은 항상 스켈레톤
5. 카드형 반복은 최대 6개, 행형은 3~5개
6. 로딩 실패 시 스켈레톤 자리를 그대로 에러 안내로 교체 (Design.Navigation 500 문구 축약형 + "다시 시도")

## 인스턴스 6종 (카탈로그 dc.html과 1:1)
팀 탐색 팀 카드(진한 커버+걸친 엠블럼 원) / 선수단 선수 카드(중앙 아바타) / 경기기록 대회·리그 행(아이콘 박스+2줄+상태 칩) / 대시보드 스탯 카드 3칸 / 알림 센터 행(원+2줄) / 공개 선수 프로필 헤더(아바타+2줄+스탯 3칸)

다른 화면은 이 6종의 골격 조합으로 파생 — 새 패턴을 만들지 말 것.
