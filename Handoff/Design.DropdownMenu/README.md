# Handoff: 드롭다운 메뉴 패턴 (Dropdown Menu)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.DropdownMenu`
> 공용 패턴 14번째. 모바일 파일 없음 — 카탈로그에 바텀시트 변환 실물 포함.

## 경계 (핵심)
- **드롭다운 메뉴 = 액션 목록**(이동·수정·삭제)만 — 값 선택은 셀렉트(FormPatterns), 조건·정렬은 칩·세그먼트·정렬(SearchFilter)
- 항목 **7개 이하** — 넘치면 화면을 나눌 신호
- 공통: 바깥 클릭/Esc 닫힘 · 동시 1개 · 열림 상태 URL 제외

## 1. 계정 메뉴 — GNB 아바타 (전 화면 공통)
- 카드: radius 13 + 보더 + 그림자 `0 10px 30px rgba(28,43,74,.12)`, max-width 240px, 우측 정렬
- 구조: 헤더(아바타+이름+역할 캡션) → 이동 그룹(대시보드 허브·설정, 아이콘 15px) → 구분선 → 로그아웃(회색 — 위험 액션 아님)
- 항목 36px, 호버 `#f8f9fb`, 12.5px 700

## 2. 오버플로 메뉴 (⋯) — 행·카드 액션
- **주 액션 1개는 행에 직접 노출**, 나머지만 ⋯(32px 버튼)로
- **파괴 액션 = 항상 맨 아래 + 구분선 + 레드 `#c0392b`** → 확인 모달(FeedbackPatterns) 연결
- 액션이 1개뿐이면 ⋯ 금지 — 직접 버튼으로
- 사용처: 선수단 행 · 일정 행 · 팀 소식 카드

## 3. 모바일 = 전부 바텀시트
- 앵커 위치 팝업 금지(엄지 도달권) — FeedbackPatterns 시트 규칙
- 구조: 그랩바 + 대상 이름 헤더(12px 회색) + 행 48px + 파괴 레드 맨 아래 + 취소 버튼(`#f1f3f8`)

## Files
- `Dropdown Menu Patterns.dc.html` — 카탈로그 3블록(계정/오버플로 토글 인터랙티브 + 바텀시트)
- `support.js` — 레퍼런스 실행용 (구현 대상 아님)

## Claude Code 첫 프롬프트 예시

```
AccountMenu(GNB)/OverflowMenu(⋯, 모바일=바텀시트) 공용 컴포넌트를 만들고

핵심: 주 액션은 직접 노출, 파괴=맨 아래+레드+확인 모달, 모바일 앵커 팝업 0건.
완료 후 선수단 행 삭제 플로우(⋯→모달→토스트)로 검수 요청해.
```
