# Handoff: 테이블 · 리스트 패턴 (Table & List)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.TableList`
> 전 페이지 공용 테이블·리스트 컴포넌트 명세. 공용 패턴 6번째.

## 원칙
1. PC에서 열 5개 이상 데이터만 테이블, 그 외는 리스트 행
2. **모바일에서 테이블은 카드 리스트로 변환** — 핵심 지표 1개만 우측 크게(+레이블), 나머지는 메타 줄로 강등("12경기 · 득실 +14"). 무엇을 버릴지가 설계
3. 가로 스크롤은 **순위표 풀버전만 예외** — "전체 기록 보기" 토글로만 진입, 순위·팀명 좌측 고정+그림자 스크롤 힌트
4. 한 행에 셰브론+버튼 혼용 금지

## 데이터 테이블 (PC — 리그 순위표)
- 헤더 11px 800 `#8a93a6`, 구분선 1.5px / 행 구분선 1px `#f4f6fa`
- 숫자열 `font-variant-numeric: tabular-nums`, 중앙 정렬 · 정렬 가능 열은 화살표(동시 1열만)
- 상위권(진출권) = teal 틴트 `rgba(46,196,182,.05)` / **내 팀 행 = 네이비 보더+틴트+"내 팀" 뱃지**(스크롤 밖이면 상단 고정 요약 1줄)
- 승/무/패 도트: teal `#2EC4B6` / 회색 `#c5cfe4` / 연레드 `#e88b7d`

## 리스트 행 3형
- **내비게이션형**: 행 전체 탭 + 우측 셰브론 (알림 센터, 일정)
- **액션형**: 우측 버튼, 행 탭 없음(오탭 방지) — 모바일은 버튼을 아래 행 풀폭 42px로. **스와이프 액션 금지**(발견 불가능 + 보호자 연령대)
- **정보형**: 스코어 tabular-nums + 결과 뱃지(승 teal / 무 회색 / 패 연레드) (경기 결과)
- 행 높이 최소: PC 44px / 모바일 56px

## 더 불러오기
- 기본 20개 + "더 보기 (N개 남음)" 버튼. **무한 스크롤 금지**(푸터 도달·URL 공유 충돌). 숫자 페이지네이션은 관리자 테이블에만

## 적용 화면
경기기록 순위표·일정 / 팀 대시보드 선수단 관리·승인 요청 / 알림 센터 / 팀 공개홈 경기 결과
