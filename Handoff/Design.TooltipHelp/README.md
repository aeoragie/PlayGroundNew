# Handoff: 툴팁 · 헬프 패턴 (Tooltip & Help)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.TooltipHelp`
> 공용 패턴 12번째. 모바일 파일 없음 — 호버 툴팁은 모바일에 부재, 팝오버·힌트는 동일 동작.

## 원칙 (핵심)
- **필수 정보는 절대 툴팁에 넣지 않는다** — 항상 보이게. 툴팁·팝오버는 보조 설명 전용
- 우선순위: ① 헬퍼 텍스트(FormPatterns — 상시 노출, 공간 있으면 항상 이것) ② 인포 팝오버(탭 열림) ③ 호버 툴팁(PC 전용)
- 모바일에 호버 없음 — 호버 툴팁에 의존하는 정보 금지

## 1. 호버 툴팁 — PC 아이콘 버튼 레이블 전용
- 다크 네이비 `#1c2b4a` 캡슐, 11.5px 700 흰 텍스트, radius 8, 하단 화살표 5px
- 지연 400ms 표시 · 이탈 즉시 소멸 · **텍스트 한 줄**(줄바꿈 필요하면 팝오버로 승격)
- 텍스트가 이미 보이는 버튼에는 금지(중복)

## 2. 인포 팝오버 — ⓘ 탭 열림 (PC·모바일 공용)
- 트리거: ⓘ 20px 회색 원(`#f1f3f8`), 레이블 옆 6px
- 탭 토글, 바깥 탭으로 닫힘 · 흰 카드 radius 12 + 그림자, max-width 300px
- 내용: 제목(12.5px 800) + 본문 2~3문장 + "자세히 보기" 링크 최대 1개
- 사용처: 서비스 고유 개념 — Claim("내 자녀예요"), 에이전트 열람 승인(30일 만료), 인증팀, Records 집계 기준

## 3. 첫 사용 힌트 — 1회성 배너
- teal 틴트(`rgba(46,196,182,.05)` + `.4` 보더) radius 12, 아이콘 원 28px + 제목+부제 + X
- **X로 닫으면 재노출 없음**(로컬 저장) · 화면당 최대 1개
- **오버레이 코치마크 투어 금지** — 보호자 연령대: 방해가 학습을 이김
- 사용처: 허브 첫 방문(자녀 연결 안내) · 팀 대시보드 첫 개설

## Files
- `Tooltip Help Patterns.dc.html` — 카탈로그 3블록(팝오버 토글·힌트 닫기 인터랙티브)
- `support.js` — 레퍼런스 실행용 (구현 대상 아님)

완료 후 모바일 뷰에서 정보 누락 없는지 검수 요청해.
