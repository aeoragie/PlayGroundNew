# Handoff: 빈 상태 카탈로그 (Empty States)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.EmptyStates`
> 이 패키지는 **전 페이지 공용 빈 상태 컴포넌트 명세**입니다. 각 화면 핸드오프에 빈 상태가 없던 자리를 이 카탈로그로 채우세요.

## 문법 2단계

### Tier A — 페이지/탭 수준 (콘텐츠 영역 전체가 빔)
- 일러스트 88px: **연한 외곽 `#c5cfe4` stroke 3 + 네이비 `#23408e` 디테일 stroke 3.5** (에러 페이지와 동일 문법)
- 제목 15.5px 800 → 설명 12.5px `#5b6577` (word-break:keep-all) → CTA(margin-top 18px)
- CTA: 소유자 액션 = 네이비 채움(40px, radius 10) / 보조 탐색 = 흰 아웃라인. **오렌지 금지**(오렌지는 랜딩 CTA 전용)

### Tier B — 카드/섹션 수준 (페이지 내 한 블록만 빔)
- 아이콘 박스 40px `#f8f9fb` radius 11 + 모노톤 라인 아이콘 19px stroke 1.8 + 한 줄 문구 13px 700 + 텍스트 링크(선택)
- 일러스트 없음 — 섹션 하나 때문에 페이지가 소란해지지 않게

## 공통 규칙
1. **CTA는 그 화면의 소유자에게만.** 방문자에게는 안내 문장으로 종료 ("팀이 영상을 올리면 이곳에서 볼 수 있어요")
2. **자동 데이터는 입력 부담 제거 문구**: 경기 기록류 = "첫 경기 결과가 반영되면 자동으로 채워져요. 따로 입력할 필요 없어요."
3. **허브 '처리가 필요해요' 예외**: 빈 상태를 그리지 않고 섹션 자체 숨김 (Design.DashboardHub 규칙)
4. 이모지 금지, 톤은 "~해요"체, 탓하지 않는 문구(검색 0건 = "필터를 넓혀보세요")

## 인스턴스 목록 (카탈로그 dc.html과 1:1)

**Tier A** (6): 팀 대시보드·선수단 0명(소유자, ＋선수 등록) / 팀 탐색·검색 결과 없음(필터 초기화) / 팀 공개홈·경기영상(방문자, CTA 없음) / 팀 공개홈·리뷰(방문자, 리뷰 남기기) / 경기기록·진행중 대회 없음(지난 시즌 아카이브) / 선수 프로필·기록 없음(CTA 없음, 자동 문구)

**Tier B** (6): 팀 대시보드·일정(＋일정 추가) / 알림 센터(문구만) / 팀 공개홈·소식 / 선수 대시보드·하이라이트(＋영상 링크 추가) / 허브·처리 필요(→숨김) / 설정·로그인 수단 1개(연결하기)

같은 성격의 새 빈 상태가 생기면 이 문법으로 파생 — 새 패턴을 만들지 말 것.
