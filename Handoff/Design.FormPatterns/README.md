# Handoff: 폼 입력 · 검증 패턴 (Form Patterns)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.FormPatterns`
> 전 페이지 공용 폼 컴포넌트 명세. EmptyStates·LoadingStates·FeedbackPatterns와 함께 공용 패턴 4부작.

## 필드 공통
- PC 높이 44px / 모바일 46px · radius 11 · 보더 1.5px `#e6e8ee` · **모바일 폰트 16px 이상**(iOS 자동 줌 방지)
- 레이블 12.5px 800 상단 고정 — **placeholder로 레이블 대체 금지**(placeholder는 예시 전용)
- 필수 표시(*) 대신 **선택 항목에만 "(선택)"** — 대부분 필수인 서비스라 역표기
- 상태 5종: 기본 / 포커스(네이비 보더 + 링 `rgba(35,64,142,.12)` 3px) / 채움 / 오류(`#c0392b` 보더 + 인라인 메시지 11.5px) / 비활성(`#f8f9fb` 배경 + 사유 문구)

## 검증 타이밍 (핵심)
1. 첫 검증은 **blur 시** — 입력 중 지적 금지
2. 오류 상태에서는 **입력 즉시 재검증** — 고치는 순간 해제
3. 제출 시 전체 검증 → **첫 오류 필드로 스크롤+포커스** (모바일은 고정 바 높이 오프셋)
4. 오류 문구 = "~해 주세요" 행동형 · **폼 오류에 토스트 금지**(FeedbackPatterns 결정표)

## 컨트롤별
- **자동 포맷**: 생년월일 `20110314 → 2011. 03. 14` / 전화 하이픈 자동 · `inputmode` 지정(numeric/tel) · 연락처류엔 "프로필에 노출되지 않아요" 프라이버시 헬퍼
- **글자수 카운터**: 레이블 우측 `86 / 200`, 초과 시 카운터만 레드(입력 차단 안 함)
- **선택지 4개 이하 = 라디오 카드**(포지션 GK/DF/MF/FW — 선택 = 네이비 보더+틴트) / 5개 이상 = 셀렉트
- **모바일 셀렉트 = 바텀시트 목록**(네이티브 금지 — 옵션 부가정보 표시)
- **체크박스**: 20px(모바일 22px) radius 6~7 네이비 채움 — 만 14세 미만 동의 등 법적 동의에 사용, 기본 체크 금지

## 제출 버튼
- 상태 3종: 기본(네이비 46~48px) / 로딩(스피너 + "저장 중…" — **스피너가 허용되는 유일한 자리**, 이중 제출 잠금) / 비활성(`#c6cede`)
- **비활성은 제출 직후에만.** 필수 미입력 상태에서도 누를 수 있게 두고 인라인 오류로 안내 — 무엇이 빠졌는지 보여주는 편이 낫다
- 모바일: **하단 고정 바**(키보드 열리면 키보드 위 부착)

## Files
- `Form Patterns.dc.html` — PC 카탈로그(4블록) / `Form Patterns Mobile.dc.html` — 모바일(390px, inputmode·고정 바 포함)
- `support.js` — 레퍼런스 실행용 (구현 대상 아님)

## Claude Code 첫 프롬프트 예시
```
Handoff/Design.FormPatterns/README.md와 두 카탈로그 dc.html을 읽어.
공용 입력 컴포넌트(TextField/Select/RadioCards/Checkbox/SubmitButton)를 만들고
기존 폼(온보딩·선수 등록·설정·Claim)에 검증 타이밍 규칙대로 연결해.
핵심: blur 첫 검증→오류 중 즉시 재검증→제출 시 첫 오류 포커스, 폼 오류 토스트 금지,
모바일 16px+inputmode+하단 고정 바.
완료 후 선수 등록 폼을 빈 제출→수정→성공 시나리오로 검수 요청해.
```
