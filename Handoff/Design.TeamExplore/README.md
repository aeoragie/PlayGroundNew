# Handoff: 팀 탐색 (Team Explore)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.TeamExplore`
> 이 번들의 HTML은 **디자인 레퍼런스(인터랙티브 프로토타입)**입니다. 코드베이스 환경(Blazor WASM + Tailwind)에서 재구현하세요. 토큰은 기존 핸드오프와 동일.

## Overview
공개 탐색 페이지 — 라우트 `/teams`. 비로그인 접근 가능. 랜딩 Phase 1과 허브 바로가기에서 링크되는 화면으로, 학부모가 **정보(핵심가치·전적·리뷰)로 팀을 비교**하는 곳. 공개 팀 홈페이지(Design.TeamPublicHome)로 보내는 유입 관문.

## 구조
- **GNB**: 서비스 페이지 규칙 — 워드마크 + 구분선 + "팀 탐색" / 우측 경기기록·로그인
- **검색 + 정렬**(추천/리뷰/선수단순) — PC 우측, 모바일은 필터 시트 안
- **필터**: 연령(전체/U12/U15/U18) · 지역 · 유형(클럽/학교/학원) pill + **인증팀만·모집중만 teal 토글**
  - PC: 2줄 필터 바 카드 / 모바일: sticky 헤더에 검색+필터 버튼(적용 개수 뱃지)+가로 스크롤 칩(연령·토글), 나머지는 펼침 시트
- **결과 헤더**: "N개 팀" + 적용 조건 요약. 비로그인 기본 문구 = "로그인하면 자녀 연령·지역 우선 정렬"
- **팀 카드** (PC grid minmax 330px / 모바일 세로 리스트):
  - 커버(64/56px): 팀 홈페이지 커버 사진(네이비 그라디언트 오버레이 `rgba(28,43,74,.08)→.35`) / 사진 없으면 폴백 — **인증팀 = 네이비 미드톤 고정, 비인증 = 파스텔** (엠블럼 주조색 추출로 자동화 가능)
  - "모집중" 오렌지 뱃지 = 커버 우상단 (카드 내 유일한 오렌지)
  - 엠블럼(52/46px, 흰 3px 보더)만 커버에 걸침 — **팀명·인증 뱃지는 커버 경계 아래 여백 확보** (padding-top으로 분리)
  - 핵심가치 teal 칩(최대 2) + 3스탯(올해 전적 / 선수단 N명 / ★평점·리뷰 수)
- **팀 획득 훅**: "우리 팀이 아직 없나요?" 네이비 카드 + 오렌지 "팀 홈페이지 만들기"

## 데이터 소스 (전부 읽기전용)
- 팀 목록: Team + 공개 홈 데이터(핵심가치·커버 사진), 시즌 전적 자동 집계, 리뷰 평점/수
- 로그인 시 개인화: 자녀 연령·지역 우선 정렬 (Records와 동일 규칙)
- 카드 클릭 → `/team/{slug}`

## Files
- `Team Explore.dc.html` — PC / `Team Explore Mobile.dc.html` — 모바일(390px)
- `support.js` — 레퍼런스 실행용 (구현 대상 아님). 커버 사진은 Pexels 스톡
