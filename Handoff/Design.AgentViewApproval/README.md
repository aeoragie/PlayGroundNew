# Handoff: 에이전트 열람 승인 (Agent View Approval — 보호자 측)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.AgentViewApproval`
> 이 번들의 HTML은 **디자인 레퍼런스(인터랙티브 프로토타입)**입니다. 코드베이스 환경(Blazor WASM + Tailwind)에서 재구현하세요. 토큰은 기존 핸드오프와 동일 (에이전트 문법 = violet `#6d4bd8`).

## Overview
보호자가 에이전트의 상세 정보 열람 요청을 심사하는 화면 — 라우트 `/approvals/agent/{requestId}`. 진입 = 알림 센터 "열람 요청" 알림(Design.ClaimFlow) 딥링크. **미성년자 보호 관문**이므로 알림 설정에서 끌 수 없는 승인형 알림(Design.Settings 규칙)과 한 쌍.

## 상태 머신 (레퍼런스 상단 프로토타입 바로 3상태 전환)
- **pending**: 승인/거절 대기. 액션 = [30일 열람 승인(teal, 주 버튼)] [거절] + "이 에이전트의 요청 다시 받지 않기"(차단)
- **approved**: 만료 카운트다운("8/16까지 · 23일 남음") + **열람 기록 로그**(에이전트가 언제 뭘 봤는지 — 신뢰의 핵심) + [열람 권한 철회](빨강 아웃라인, 즉시 revoked→denied 동급)
- **denied**: "사유 없이 거절만 전달" 안내 + 동일 에이전트 30일 재요청 제한. 요청 기록에서 재승인 가능

## 화면 구성
### 공통 콘텐츠 블록
1. **에이전트 신원 카드** (violet 보더 `rgba(109,75,216,.3)` + 헤더 배경 `.06`): 아바타 + 이름 + "인증 에이전트" 뱃지 + 소속·등록연도 + 3스탯(중개 이력 / ★평가 / 활동 지역) + 프로필 전체 보기 링크. **인증 에이전트만 요청 가능**(비인증은 요청 버튼 자체가 없음 — 에이전트 서비스 측 규칙)
2. **요청 내용**: 에이전트가 쓴 메시지(선수명 굵게) + 요청 시각
3. **승인 시 열람되는 정보** 체크리스트: teal 체크 4항목(경기별 상세 기록 / 학교·학년 / 커리어 검증 상태 / 보호자에게 연락) + 빨강 X **"연락처 직접 노출 — 항상 제외, 플랫폼 메시지만"** (고정 문구, 변경 금지)
4. **안전 안내** (teal 배경 박스): 30일 자동 만료 · 열람 기록 표시 · 언제든 철회

### 레이아웃
- **모바일**(`Agent View Approval.dc.html`, 420px 카드): 세로 스택 — 신원→요청→범위→액션
- **PC**(`Agent View Approval PC.dc.html`, max 980px): 대시보드 GNB + 브레드크럼("알림 센터 · 열람 요청") + 선수명 포함 H1 + 2컬럼(좌 1.15fr = 신원+요청+열람 기록 / 우 .85fr sticky = 범위+액션)

## 백엔드 연결점
- ViewGrant { agentId, playerId, guardianId, status, expiresAt(승인+30d), scopes(고정 세트) }
- 승인/철회 즉시 공개 선수 프로필(Design.PlayerPublicProfile)의 "디테일 권한 뷰" 접근 갱신
- ViewLog: 에이전트의 열람 이벤트 적재 → approved 상태 화면에 노출
- 차단: AgentBlock { guardianId, agentId } — 요청 생성 자체를 거부
- 거절 후 30일 재요청 쿨다운은 서버 강제

## 완료 기준 체크리스트
- [ ] 3상태 렌더링 + 전이(승인→approved, 거절/철회→denied) 정확
- [ ] 열람 범위 = 고정 세트(사용자 선택 없음), 연락처 제외 문구 그대로
- [ ] 만료 자동 처리(30일) + 남은 일수 표시, 열람 기록 시간순
- [ ] 승인/철회가 선수 프로필 권한 뷰에 즉시 반영
- [ ] violet은 에이전트 요소에만, 주 버튼 = teal(오렌지 아님 — CTA 아닌 승인 동작), 이모지 0개, 한글 줄바꿈 결함 없음
