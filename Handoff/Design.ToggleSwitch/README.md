# Handoff: 토글 · 스위치 패턴 (Toggle & Switch)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.ToggleSwitch`
> 공용 패턴 9번째. 별도 모바일 파일 없음 — 스위치 크기 동일(44×26), 모바일 차이는 "터치 타겟 = 행 전체 56px" 한 줄. 체크박스·라디오는 FormPatterns, 세그먼트는 SearchFilter 참조.

## 사용 결정표 (핵심 — 역할 혼용 금지)
- **스위치** = 즉시 적용되는 켬/끔 (설정 화면 — 저장 버튼 없는 화면 전용): 알림 수신·프로필 공개
- **체크박스** = 폼 안의 동의·다중 선택 (제출로 저장): 약관 동의·관심 지역
- **라디오 카드** = 폼 안의 상호배타 ≤4개: 포지션·역할 선택
- **세그먼트** = 화면 데이터의 축 전환 (항상 1개): 대회/리그·연령대
- 경계 규칙: **폼 안(저장 버튼 있는 화면)에 스위치 금지** — 즉시 적용처럼 오해됨. 설정 켬/끔에 체크박스 금지.

## 스위치 명세
- 44×26px, knob 22px 흰 원+그림자, radius 99 · 켬 = teal `#2EC4B6` / 끔 = `#d5d9e2`
- 상태 5종: 켬 / 끔 / **잠금** = 스위치 대신 "항상 켜짐" 뱃지(승인형 알림 — Settings 규칙, 서버에서도 강제) / **비활성** = 50% 투명 + 사유 캡션 필수 / 저장 중 = **낙관적 반영 → 실패 시 롤백 + 오류 토스트**
- 레이아웃: 레이블(13px 700)+설명(11px) 좌측, 스위치 우측 · 행 전체 탭 가능(모바일 56px)

## 계층 스위치 (선수 프로필 공개 범위)
- 상위(프로필 공개) 끄면 하위 그룹(경기별 상세·학교 표시) **접힘+비활성**(들여쓰기 16px, 좌측 2px 보더)
- 끄기는 파괴적이지 않음 — 모달 없이 즉시 + 실행취소 토스트 (FeedbackPatterns)

## 적용 화면
설정 알림(스위치+잠금) · 선수 프로필 공개 범위(계층) · 온보딩 동의(체크박스) · 선수 등록 포지션(라디오 카드) · 경기기록(세그먼트)
