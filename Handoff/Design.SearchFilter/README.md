# Handoff: 검색 · 필터 패턴 (Search & Filter)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.SearchFilter`
> 전 페이지 공용 검색·필터 컴포넌트 명세. 공용 패턴 5번째 (EmptyStates·LoadingStates·FeedbackPatterns·FormPatterns에 이어).

## 핵심 원칙
1. **필터는 즉시 적용** — 적용 버튼은 모바일 바텀시트 안에서만(지연 적용)
2. **상태는 URL 쿼리 동기화** — 공유·뒤로가기·새로고침에도 유지 (`/teams?region=seoul&age=u15`)
3. **결과 카운트 항상 표시**, 필터 반영 즉시 갱신
4. 0건 = Design.EmptyStates "검색 결과 없음" + 필터 초기화 / 로딩 = 스켈레톤 유지(레이아웃 점프 방지)

## 컴포넌트별
- **검색 바**: 디바운스 300ms 실시간 · 입력 시 지우기(X) · Esc=비우기 · 초성·부분 일치 · 모바일 폰트 16px
- **필터 칩**(다중, 좁히기 0~N개): 34px(모바일 36px) radius 99 · 선택=네이비 채움+X 포함 · "전체"는 상호배타
- **세그먼트**(축 전환, 항상 1개): `#eef0f4` 트랙+선택=흰 카드+그림자. 칩과 역할 혼용 금지 — 세그먼트=데이터 축(대회/리그, 연령), 칩=조건 좁히기
- **결과 헤더**: "팀 **24**개"(카운트 네이비) · "필터 초기화"(필터 ≥1일 때만 노출) · 정렬 드롭다운(기본값 화면별 고정: 탐색=추천순, 기록=최신순)
- **GNB 퀵서치**(팀·선수 전역): PC=드롭다운 그룹핑(팀/선수 각 최대 3건+"전체 결과 보기"), 일치 부분 네이비 하이라이트, ↑↓ 키보드 / 모바일=풀스크린 전환(뒤로가기=닫기), 최근 검색어 3개 로컬 저장. 결과 행=이름/메타 2줄 구조(각각 nowrap)

## 모바일 전용
- 칩 행 가로 스크롤(스크롤바 숨김) + 맨 앞 **"필터" 버튼 = 적용 개수 뱃지**
- 필터 바텀시트: 그랩바+"초기화"+섹션별 칩+하단 **[N개 팀 보기]**(결과 수 실시간) — 시트 안에서만 지연 적용
- 터치 타겟: 칩 36px, 시트 버튼 48px

## 적용 화면
팀 탐색(검색+지역·연령·모집중·인증팀 칩+정렬) · 경기기록(연령 세그먼트+대회/리그 세그먼트+상태 칩) · 팀 대시보드 선수단(검색+포지션 칩) · GNB 퀵서치(전역)
