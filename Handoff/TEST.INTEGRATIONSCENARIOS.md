# TEST.INTEGRATIONSCENARIOS — 통합 테스트 시나리오

> 위치: `Handoff\TEST.INTEGRATIONSCENARIOS.md` · 기준: Phase A~C 전체 + FriendlyMatch·RecordCorrection (2026-07-21)
> 사용법: 시나리오 단위로 Claude Code에 지시하거나 직접 수행. 각 시나리오는 계정 준비 → 여정 → 확인 항목(✓) 순. **여정 중간의 상태(로딩 스켈레톤·빈 상태·토스트)도 확인 대상.**

## 준비 계정 4종
- **T** 팀 관리자 (FC 한강 — 선수 10+, 경기 기록 있음)
- **P** 보호자 (자녀 2: 연결됨 1 + 미연결 1)
- **N** 무소속 신규 회원 (역할 미설정 General)
- **G** 게스트 (비로그인)
- 보조: **E** 빈 팀 관리자 (EmptyFC — 빈 상태 검증용)

---

## S1. 게스트 → 가입 → 역할 분기 (G→N)
랜딩 진입 → 히어로 보조 CTA "우리 동네 팀 둘러보기" → 팀 탐색(게스트 열람) → 팀 카드 → 공개 팀 홈 6탭 → GNB [로그인] → 소셜 가입 → General 상태로 `/dashboard` 진입
- ✓ 게스트 GNB(모바일=워드마크+로그인만) · 팀 탐색 필터 URL 공유 재진입 유지
- ✓ `/dashboard` 3분기: General → 역할 선택 화면
- ✓ returnUrl: 로그인 전 보던 팀 홈으로 복귀

## S2. 보호자 여정 — Claim → 허브 → 기록 열람 (P, 모바일 우선)
로그인 → 역할 2개 이상 → 허브 표시 → "＋ 자녀 연결" → Claim 4스텝(스텝퍼: 이탈 후 복귀 시 입력 유지) → 신청 완료 → T 계정에서 승인 → P 허브 자녀 카드 반영
- ✓ 허브: Pending 자녀 = 스탯 "–" + 공개 프로필 버튼 미노출 → 승인 후 정상 카드
- ✓ 알림: 승인 시 P 알림 센터 발송 + 벨 카운트(허브와 동기, 0이면 숨김)
- ✓ 스텝퍼: 완료 스텝만 클릭 이동 / 모바일 = 진행 바
- ✓ 자녀 공개 범위 설정: 상위 스위치 끔 → 하위 접힘, 공개 프로필에서 즉시 반영

## S3. 팀 관리자 여정 — 팀 운영 풀사이클 (T)
로그인(역할 1개면 허브 스킵 확인) → 팀 대시보드 → 선수 등록(폼 검증: 빈 제출→첫 오류 포커스) → 선수 사진 업로드(12MB 실패 인라인 → 재선택 성공) → 일정 추가(캘린더: 과거 비활성·시간 기본값=최근 일정) → 친선경기 결과 입력 → 팀 정보 수정(엠블럼 교체) → 공개 홈 반영 확인
- ✓ 친선 결과: 순위표 미반영 · 시즌 스탯 "+N" 별도 · 공개 프로필 미노출 · 행 점선 스타일
- ✓ 삭제 플로우: 선수 삭제 ⋯→확인 모달(포커스=취소)→토스트 실행취소 왕복
- ✓ 엠블럼 삭제 시 이니셜 복귀(네이비)

## S4. 기록 수정 신청 왕복 (T 또는 P)
공식 경기 행 ⋯ "기록 수정 신청" → 폼(현재→올바른 대비, 1건 1항목) → 제출 → 같은 경기 재신청 시도(차단 확인) → 접수 상태에서 취소 → 재신청 → (대회 서비스 대신 DB 직접) 반려 처리 → 사유 표시·알림 확인
- ✓ 친선 행에는 신청 메뉴 없음 · 미처리 중 "신청 처리 중" 비활성
- ✓ 반려 사유 박스 + 알림 센터 발송

## S5. 검색·탐색 축 (전 계정)
GNB 퀵서치(팀·선수 그룹핑·하이라이트·모바일 풀스크린) → 팀 탐색 필터 조합(칩 즉시 적용·카운트 갱신·"전체" 상호배타) → 0건 만들기 → 빈 상태+필터 초기화 → 모바일 바텀시트(지연 적용 "[N개 보기]")
- ✓ 뒤로가기·새로고침·URL 공유 3종 모두 필터 유지

## S6. 권한·경계 침투 테스트
- G가 대시보드 URL 직접 진입 → `/auth?returnUrl` / P가 남의 팀 대시보드 URL → 403(실드 일러스트)
- 없는 라우트 → 404 / 강제 예외 → 500(오류 코드 표기)
- 제3자 계정에서 선수 사진 업로드 버튼 **미노출**(비활성 아님) 확인
- 에이전트 열람 승인: 요청→P 승인→30일 만료·연락처 항상 제외·열람 로그

## S7. 빈 계정 스모크 (E + N)
E로 전 화면 순회: 대시보드·공개 홈·기록 — 빈 상태(Tier A/B, CTA는 소유자만) 노출, 레이아웃 붕괴 없음. 네트워크 스로틀로 스켈레톤(200ms 지연·점프 0) 확인.

## 공통 확인 (모든 시나리오에 겹쳐서)
1. 이모지 0 · 오렌지는 랜딩 CTA/카운트/팀 카드 주 버튼만
2. 한글 줄바꿈 결함 0(버튼·메타 nowrap)
3. 모바일 390px: 터치 44px+, 입력 16px(줌 0), 바텀시트 대체
4. 폼 오류 토스트 0건(전부 인라인) · 오류 토스트는 자동 소멸 안 함
5. 콘솔 에러 0 · 죽은 링크 0(배선표 기준)

## 실행 순서 권장
S1 → S3 → S2 → S4 (데이터가 앞 시나리오에서 쌓이는 순서) → S5·S6·S7은 병렬 가능.
