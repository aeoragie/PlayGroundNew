# Handoff: 설정 (Settings)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.Settings`
> 이 번들의 HTML은 **디자인 레퍼런스(인터랙티브 프로토타입)**입니다. 코드베이스 환경(Blazor WASM + Tailwind)에서 재구현하세요. 토큰은 기존 핸드오프와 동일.

## Overview
계정 수준 설정 — 라우트 `/settings/{section}`. 3섹션: 계정 / 역할 / 알림.
설계 원칙: **알림 센터(수신함)와 알림 설정(정책)은 분리** — 센터는 대시보드 벨, 정책은 여기.

## 구조
- **PC**: 대시보드 GNB(네이비, 워드마크+구분선+"설정", 우측 "대시보드로") + 좌측 sticky 사이드 메뉴 210px(계정/역할/알림 + 구분선 + 로그아웃 빨강) + 콘텐츠
- **모바일**: 상단 바(뒤로가기+"설정") + **세그먼트 탭 3개**(teal 언더라인) + 세로 스택. 로그아웃 = 계정 탭 하단 풀폭 버튼

## 섹션별
### 계정
- 프로필 행: 아바타 + 이름 + 마스킹 이메일(kim***@) + "이름 변경"
- **연결된 로그인**: Google(공식 4색 아이콘) "연결됨" teal 뱃지 / Kakao(공식 옐로) 점선 + "연결하기". LINE도 같은 패턴(Design.Auth.Onboarding 참조)
- 계정 관리: 데이터 내려받기(개인정보보호법 대응) / **계정 삭제**(빨강 `#c0392b`, 보더 `#f2d5d0`) — 캡션 "자녀 프로필은 다른 보호자에게 이전하거나 함께 삭제돼요" 필수(FamilyLink 이전 플로우와 연결)

### 역할
- 보유 역할 카드: 보호자(자녀 목록 요약) / 팀 관리자(팀·직책) — "사용 중" teal 뱃지
- **에이전트**: 점선+opacity .75+"준비 중" — 캡션 "별도 서비스에서 운영돼요" (서비스 분리 원칙, feature flag)
- "＋ 역할 추가" 점선 → 역할 선택 화면(Design.Auth.Onboarding) 재사용
- 역할 제거는 P0 미제공 (팀 소유권 이전·자녀 연결 해제가 선행돼야 함 — 고객센터 경유)

### 알림
- **수신 방법**: 앱 푸시 / 이메일 — teal 스위치(44×26px, knob 22px)
- **알림 항목**: 유형 칩(승인=오렌지톤 / 경기·모집·리뷰=네이비톤 / 열람=violet) + 스위치
- **핵심 규칙: "승인" 유형(연결 요청·열람 요청)은 스위치 없이 "항상 켜짐" 잠금 뱃지** — 미성년자 보호 관문이므로 끌 수 없음. 캡션 필수
- 기본값: 경기·모집·리뷰 ON, 방문 요약 OFF, 이메일 OFF

## 백엔드 연결점
- NotificationPreference(채널×항목 매트릭스, 승인 유형은 저장 불가·항상 true)
- 소셜 연결: 계정당 복수 IdP 링크, 마지막 1개는 해제 불가
- 계정 삭제: FamilyLink 이전 or 동반 삭제 확인 2단계 모달(레퍼런스엔 버튼만)
