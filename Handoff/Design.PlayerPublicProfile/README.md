# Handoff: 공개 선수 프로필 4뷰 (Player Public Profile)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.PlayerPublicProfile`
> 이 번들의 HTML은 **디자인 레퍼런스(인터랙티브 프로토타입)**입니다. 코드베이스 환경(Blazor WASM + Tailwind)에서 재구현하세요. 토큰은 기존 핸드오프와 동일.

## Overview
개발 로드맵 4단계의 공개면. 라우트 `/player/{slug}` — 선수 대시보드 데이터의 읽기전용 조합.
**4뷰 = 2축 조합**: 디테일/카드 × 공개/권한 열람. 레퍼런스 상단의 네이비 뷰 전환 바는 **프로토타입 전용** — 실제로는 뷰어의 계정·승인 상태로 자동 결정 (구현 금지).

| 뷰 | 대상 | 내용 |
|---|---|---|
| 디테일 · 공개 | 누구나(비로그인) | 공개 항목만 + 잠금 안내 + "상세 정보 열람 요청" CTA |
| 디테일 · 권한 | 보호자 승인 계정(에이전트·지도자) | + 학교 칩, 경기별 상세 기록, "보호자에게 연락하기" |
| 카드 · 공개 | SNS·메신저 공유 | 요약 카드(공개 정보만) + QR + 이미지 저장 |
| 카드 · 권한 | 승인 계정 | + 승인 열람 블록(학교·보호자·연락 수단), 재공유 금지 워터마크 |

## 보호자 통제 원칙 (전 뷰 공통 — 필수 구현)
- 공개 항목은 선수 대시보드의 PlayerFieldVisibility를 그대로 따름 (기본: 키·몸무게·주발 공개 / 학교·연락처 비공개)
- 열람 요청 → 보호자 알림 → 승인(30일 만료) → **열람 기록 전부 보호자에게 표시** (배너에 명시)
- 연락은 항상 플랫폼 내 메시지로 시작 — 전화번호 직노출 금지
- 권한 카드: 워터마크 + 재공유 제한 문구

## GNB (플랫폼 서비스 페이지 규칙)
- 좌측: **텍스트 전용 워드마크** "PlayGround"(네이비 #23408e, 800) + "Soccer"(연회색 #b6bdc9, 700) — 아이콘·이모지 금지. PC는 세로 구분선 + "선수 프로필" 레이블(13px 700 #3c465a)
- 우측: 경기기록 · 로그인
- (참고: 팀 홈페이지는 반대로 팀 엠블럼이 좌측 — 서비스 페이지와 팀 자체 사이트의 구분)

## 구조
- **PC**: 디테일 = 히어로 카드(3:4 사진 150px + 이름·뱃지·소속·공개 칩·태그) → 시즌 4카드 → [대표 영상 | 커리어 380px] → 권한: 경기별 기록 테이블 / 공개: 잠금 안내 박스. 카드 = 360px 세로 카드 중앙 배치
- **모바일**: 세로 스택 + **하단 고정 CTA 바**(공유 아이콘 48px + 오렌지 풀폭, safe-area). 카드 = 340px

## 선수 카드 디자인
네이비 그라디언트(170deg #1c2b4a→#23408e) + 장식 원 2개 + 팀 크레스트·시즌 + #등번호(반투명 대형) + 3:4 사진 + 이름·포지션·주발·출생연도 + 4스탯 칩(출전/득점(teal)/도움/시간) + 하단 QR 바(URL·PlayGround 워터마크). 이미지 저장 = 1080×1350.
