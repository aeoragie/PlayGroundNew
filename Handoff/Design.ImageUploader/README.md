# Handoff: 이미지 업로더 패턴 (Image Uploader)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.ImageUploader`
> 공용 패턴 8번째. 별도 모바일 파일 없음 — 모바일 차이는 아래 "모바일" 절 3줄이 전부(그리드는 동일 구조로 좁아짐).

## 용도별 3형
1. **아바타/엠블럼 (원형 1:1)** — 팀 설정·선수 프로필·온보딩(건너뛰기 가능)
   - 우하단 카메라 뱃지(30px 흰 원+보더) 탭 → 파일 선택 → **원형 크롭(이동·확대만, 회전 없음)**
   - 삭제 시 이니셜/실드 placeholder 복귀 — **빈 이미지 상태 없음**
   - 업로드 후 컨트롤: "교체"(네이비 텍스트) / "삭제"(레드 텍스트)
2. **커버 (와이드 3:1)** — 팀 공개홈·팀 탐색 카드에 노출
   - 빈 상태 = 점선 드롭존(`#c5cfe4` dashed, `#fbfcfe` 배경): 아이콘 + "끌어다 놓거나 파일 선택" + 권장 크기(1600×533)·형식·용량
   - 업로드 후 = 우하단 반투명 캡슐(`rgba(28,43,74,.65)`) "교체 / 위치 조정" — 위치 조정은 **세로 드래그만**(3:1 크롭 고정)
3. **다중 사진 (그리드, 최대 10장)** — 팀 소식 작성
   - 첫 장 = 자동 대표("대표" 뱃지), 드래그 순서 변경(첫 위치가 대표)
   - 타일: 우상단 X 삭제 · 업로드 중 타일 내 스피너+% · 추가 타일 "＋ 3/10" 카운트
   - 초과 선택 시 앞 10장만 업로드 + 토스트 안내

## 상태 규칙
- 업로드 중: 진행 바(파일명+%) 또는 타일 내 진행. 완료 후 토스트 불필요(이미지가 보이는 게 완료 신호)
- **실패는 인라인**(토스트 아님 — 자리 유지가 복구에 유리): 레드 카드에 원인+해결("파일이 12MB예요 — 10MB 이하로") + "다시 선택"
- 사전 방지: **클라이언트 리사이즈 후 업로드**(장변 2000px) · EXIF 회전 보정 · jpg/png/webp · 10MB

## 권한 (미성년자 보호)
- 업로드 주체 = 소유자만: 선수 사진은 **보호자·팀 관리자만**, 방문자 업로드 경로 없음
- 신고 경로는 공개 프로필 명세(Design.PlayerPublicProfile) 참조

## 모바일
1. 드롭존 없음 — "사진 올리기" 버튼으로 대체
2. 카메라/앨범 선택은 OS 시트에 위임(input capture) — 자체 시트 만들지 말 것
3. 크롭 화면만 자체 제공(핀치 확대·드래그 이동), 풀스크린

## Files
- `Image Uploader Patterns.dc.html` — 카탈로그 4블록(아바타/커버/다중/상태)
- `support.js` — 레퍼런스 실행용 (구현 대상 아님)
