# Handoff: 페이지네이션 · 브레드크럼 패턴 (Pagination & Breadcrumb)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.PaginationBreadcrumb`
> 공용 패턴 13번째. 모바일 파일 없음 — 뒤로 헤더가 곧 모바일 형태, 숫자 페이지네이션은 관리자(PC) 전용.

## 결정표 (핵심)
- 사용자 목록 로딩 = **"더 보기"** (TableList — 20개 단위, 무한 스크롤 금지)
- 관리자 테이블 = **숫자 페이지네이션** (유일한 사용처 — 두 방식 혼용 금지)
- 위치 표시: PC 깊이 3단 이상 = 브레드크럼 / 모바일 = 뒤로 헤더 / 2단 이하 = GNB+탭으로 충분(상시 브레드크럼은 소음)

## 브레드크럼 (PC, 깊이 3단 이상만)
- 조상 = 12px 700 `#8a93a6` 링크 / 현재 = 800 `#1c2b4a` **링크 아님** / 구분 = 셰브론 11px `#c5cfe4`
- 4단 초과 시 중간 "…" 축약(첫 단 + 마지막 2단 유지) · 긴 팀·선수명 말줄임 160px
- 사용처: 경기기록 상세(기록 → 대회 → 경기). 팀 공개홈·대시보드 금지(탭이 위치 표시)

## 뒤로 헤더 (모바일 상세 화면 전부)
- ← 아이콘(터치 44px) + 제목 13.5px 800 말줄임 + 상위 맥락 1줄(10.5px `#8a93a6`)
- ←는 history.back이 아니라 **구조상 상위로** — 딥링크 진입 대비(공유 링크로 들어와도 상위로 이동 가능)

## 숫자 페이지네이션 (관리자 테이블 전용)
- 버튼 34px radius 9 · 현재 = 네이비 채움 / 나머지 = 흰+보더 · 이전/다음 셰브론(끝에서 비활성 40%)
- 7페이지 초과 시 `1 2 3 … 12` 축약 · 페이지는 URL 쿼리(`?page=3`)
