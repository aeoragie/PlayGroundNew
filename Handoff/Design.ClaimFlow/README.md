# Handoff: Claim 플로우 + 알림 센터 (Claim Flow)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.ClaimFlow`
> 이 번들의 HTML은 **디자인 레퍼런스(인터랙티브 프로토타입)**입니다. 코드베이스 환경(Blazor WASM + Tailwind)에서 재구현하세요. 토큰은 기존 핸드오프와 동일.

## Overview
GTM 플라이휠의 핵심 관문. 팀이 등록한 Unclaimed 선수 프로필을 보호자 계정과 연결(Claim)하는 전 과정 + 승인이 오가는 **알림 센터**(팀·선수 대시보드 공통 패널).
상태 흐름: **Unclaimed → (초대코드 입력 + 연결 요청) → Pending → (팀 관리자 승인) → Claimed**.

**모바일 우선(390~460px)** — 학부모가 카톡·문자로 초대코드를 받아 진입하는 시나리오가 절대다수. PC는 동일 카드 중앙 배치로 충분.

레퍼런스 상단 "시점 전환" 바와 "(프로토타입) →" 버튼은 **프로토타입 전용 — 구현 금지**. 실제로는 계정 역할로 화면 결정.

## 보호자 플로우 (라우트 `/claim`)
1. **초대코드 입력** — 6자리 코드 박스(자동 포커스 이동, 대문자 정규화). 하단 "코드가 없어요" 대안 3경로: 팀에 요청 / Records의 선수 프로필에서 직접 연결 요청 / 프로필 직접 생성
2. **프로필 확인** — 코드로 조회된 선수 카드(이름·포지션·등번호·생년·팀 + "미연결" 뱃지) + 관계 선택(어머니/아버지/기타 보호자) + 권한 이전 안내 문구(원문 유지: "승인되면 프로필 관리 권한이 보호자에게 넘어오고, 팀은 더 이상 정보를 수정할 수 없어요")
3. **승인 대기 (Pending)** — 요청 요약 카드(요청자·관계·시각 + "승인 대기" 오렌지톤 뱃지) + 기대치 문구("보통 1~2일")
4. **연결 완료 (Claimed)** — teal 체크 + 다음 행동 2가지(항목별 공개 설정 / 대표 영상 등록) + "선수 대시보드로 가기" CTA

## 알림 센터 (컴포넌트, GNB 벨에서 열림)
- 헤더: 벨 + "알림" + 미읽음 카운트(오렌지) + 컨텍스트(팀명)
- 필터 칩: 전체 / 유형별(개수 표기)
- **액션형 알림** (연결 요청): 아이콘 원(오렌지톤) + 제목 + 본문(요청자·선수·사용 코드) + **승인(teal)/거절 버튼 인라인** + 권한 이전 안내. 처리 후 완료 상태로 교체(teal 박스)
- **이동형 알림** (모집 지원자, 에이전트 추천 violet 뱃지): 셰브론 → 해당 화면 딥링크
- 보호자 측 알림도 동일 문법: 초대코드 도착, 승인 완료, 열람 요청, 승인 만료(30일)

## 백엔드 연결점
- PlayerLinkCode: 팀 대시보드에서 발송·재발송, 만료·1회성 정책 필요
- FamilyLink 생성(관계 포함) → 상태 전이 Unclaimed/Pending/Claimed → 팀·선수 대시보드 뱃지와 동일 소스
- 승인 시 권한 이전: 프로필 필드 쓰기 = 보호자, 팀은 소속·등번호만
- 알림: Notification(Type, TargetRoute, ActionPayload) — 액션형은 인라인 처리 API
- Records "내 아이 프로필 관리하기" CTA → 이 플로우의 2단계로 직행(코드 없이 연결 요청, 팀 승인 필수)
