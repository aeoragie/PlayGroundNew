# SPEC — Claim 플로우 + 알림 센터 (모바일 우선)

공통: 카드 `#fdfdfc` radius 20 + shadow `0 12px 40px rgba(28,43,74,.12)`, max-width 420px 중앙 · 배경 `#e9ecf2`(독립 플로우 화면) · 한글 nowrap/keep-all · 이모지 금지(모노톤 SVG stroke 1.8) · 워드마크 PlayGround(네이비)+Soccer(연회색).
**카피는 레퍼런스 원문 그대로. 오렌지 CTA = 화면당 1개.**

## 상태 모델
Unclaimed(회색) → Pending(오렌지톤 `#fff0e9`/`#c24a1c`) → Claimed(teal `rgba(46,196,182,.12)`/`#1b8579`) — 팀·선수 대시보드 뱃지와 동일 토큰.

## 보호자 스텝별
### ① 초대코드
- 타이틀 "초대코드를 입력해 주세요" + 부제
- 코드 박스 6칸: 46×56px radius 12, 입력됨=네이비 보더+`#f1f3f8` 배경, 미입력=회색 보더. 자동 포커스 이동, 붙여넣기 지원
- CTA "프로필 찾기"(오렌지 50px) — 코드 불일치 시 인라인 에러("코드를 다시 확인해 주세요")
- "코드가 없어요" 섹션(점선 상단 보더): 대안 3경로 리스트+링크

### ② 프로필 확인
- 타이틀 "이 프로필이 맞나요?" + "FC 한강 유나이티드가 등록한 선수 정보예요."
- 선수 카드: 이니셜 아바타 54px 네이비 + 이름·메타 2줄 + "미연결" 회색 뱃지
- 관계 선택 3버튼(선택=네이비 채움) — FamilyLink.Relation
- 안내 박스(`#f8f9fb`): 권한 이전 문구 **원문 필수**
- CTA "연결 요청 보내기" + 보조 "내 자녀가 아니에요 — 코드 다시 입력"(텍스트 버튼, 스텝 ①로)

### ③ 승인 대기
- 시계 아이콘 원(64px 오렌지톤) + "승인을 기다리고 있어요" + 기대치 문구
- 요청 요약 카드: 아바타 + "김민준 · 보호자(어머니)" + 요청 시각 + "승인 대기" 뱃지
- 이 화면은 재방문 시에도 표시(요청 취소 링크는 P1)

### ④ 연결 완료
- teal 체크 원 + "김민준 선수와 연결됐어요" + 부제
- 다음 행동 카드 2개(아이콘+설명): 항목별 공개 설정 / 대표 영상 등록 → 각각 선수 대시보드 딥링크
- CTA "선수 대시보드로 가기"

## 알림 센터
- 패널: 네이비 헤더(벨+알림+카운트 오렌지 뱃지+팀명) + 필터 칩(전체 N/연결 요청 N/모집 N, 선택=네이비 채움)
- **액션형(연결 요청)**: 배경 `#fffdfb`(미처리 강조) · 아이콘 원 38px 오렌지톤(사람+플러스) · 제목 13.5px 800 · 본문(요청자 관계·선수·포지션) · 메타(사용 코드·경과 시간) · [승인 teal 채움 | 거절 아웃라인] 42px · 하단 권한 안내 11px
- 승인 후: 버튼 영역 → teal 완료 박스("승인 완료 — … Claimed 상태가 됐어요")로 교체, 카드 배경 일반화
- **이동형**: 아이콘 원(유형색: 모집=네이비톤, 에이전트=violet `rgba(109,75,216,.1)`) + 제목/본문/시간 + 우측 셰브론, 읽음=opacity .85
- 에이전트 관련 알림엔 violet "에이전트" 뱃지 병기
