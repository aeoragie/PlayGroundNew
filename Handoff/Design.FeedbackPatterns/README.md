> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.FeedbackPatterns`
> 전 페이지 공용 피드백 컴포넌트 명세. Design.EmptyStates·LoadingStates와 3부작 — 로딩/빈/피드백.

## 사용 결정표 (핵심)
- 저장·복사·전송 **완료** = 토스트
- 삭제·공개 전환·승인 등 **되돌릴 수 없는 일의 사전 확인** = 모달 → 완료 토스트. 그 외 모달 금지
- **폼 검증 오류 = 필드 인라인** (토스트 금지)
- 서버 오류 = 오류 토스트 + 재시도
- 화면 내 변화가 눈에 보이면 토스트 생략 · 알림 센터와 중복 발송 금지

## A. 토스트
- 캡슐: 다크 네이비 `#1c2b4a`, radius 13, shadow `0 8px 24px rgba(28,43,74,.25)`, 최대 340px
- 위치: PC 좌하단(24px 마진) / 모바일 하단 중앙(탭바 위 12px). 슬라이드 업+페이드, **동시 1개(교체식)**
- 구성: 상태 원 26px(성공 teal `#2EC4B6` ✓ / 오류 `#c0392b` ! / 정보 `#23408e` i) + 메시지 13px 600 흰색 + 액션 최대 1개(teal 텍스트 버튼)
- 지속: 기본 3.5초 / 액션 있으면 5초 / **오류는 자동 소멸 안 함**(수동 닫기·재시도)
- 문구: "~됐어요" 완료형, 오류는 원인보다 다음 행동("다시 시도해 주세요")

## B. 확인 모달
- 오버레이 `rgba(28,43,74,.45)` · 카드 radius 18, 최대 420px · Enter=주 버튼, Esc=취소
- **일반**(네이비 주 버튼): 동사형 레이블("공개하기" — "확인" 금지) + 결과 설명 1줄
- **파괴적**(레드 `#c0392b`): 제목에 대상 이름 명시("김민준 선수를…") + "되돌릴 수 없어요" + **기본 포커스=취소** + 삭제 후 실행취소 토스트 이중 안전망
- **고위험 입력 잠금**(계정·팀 삭제 전용): "삭제합니다" 입력 일치 전 버튼 비활성(`#e8b0a8`) + 영향 범위 명시("자녀 프로필 2개가 함께 삭제돼요")
- **모바일 = 전부 바텀시트**: 그랩바 36×4px, 버튼 세로 스택 46px, **파괴 버튼이 위**, 취소가 아래(배경 흰색)
