# Handoff: 친선경기 구분 (Friendly Match Distinction)

> 대상 저장소: `C:\Workspace\PlayGroundNew` · 압축 해제 위치: `Handoff\Design.FriendlyMatch` (신규)
> 배경(7/19 확정): 공식 경기기록 = 주최측 기록 원칙(선수·팀 읽기 전용, 오류는 수정 신청 — 추후 설계). 기존 B1 결과 입력은 **친선경기 기록용으로 전환**해 유지.

## 핵심 원칙
1. **공식 = 기본 형태 그대로, 친선에만 마킹을 얹는다** — 공식에 "공식" 뱃지 금지(다수가 공식, 소수만 마킹)
2. **집계 경계: 순위표·시즌 스탯·공개 선수 프로필 = 공식만.** 친선은 집계 제외 + 별도 표기, 공개 프로필엔 미노출
3. 친선/연습 구분 없음 — 단일 "친선경기" (MatchType = Official/Friendly 2값)

## 명세
- **결과 행(친선)**: 점선 보더 `#d5d9e2` dashed + 배경 `#fbfcfd` + 대회명 자리에 "친선경기" 회색 라벨(공 아이콘 11px). 승패 뱃지 유지하되 **중립 회색 톤**(공식의 teal/레드와 구분 — 행당 뱃지 1개 규칙: 구분은 행 스타일+메타로, 뱃지 추가 아님)
- **세그먼트**: 결과 목록 상단 "전체 / 공식 / 친선경기" (TabAccordion 축 전환 규칙, URL 동기화)
- **스탯 카드**: "시즌 득점 (공식) 7" + "친선경기 별도 +3"(회색, 합산 금지)
- **B1 입력 화면**: 제목 "친선경기 결과 입력"(구 "결과 입력") + 부제 "대회·리그 공식 기록은 주최측이 입력해요. 기록에 오류가 있다면 수정 신청으로 알려 주세요." · 경기 성격 선택 필드 없음(전부 Friendly 저장) · **대회/리그 선택 필드 제거**
- **데이터**: Match.MatchType(Official/Friendly) — 기존 B1 저장분은 Friendly로 마이그레이션. 순위 재계산 Usp는 Official만 집계하도록 수정

## 적용 화면
팀 대시보드 경기 결과 · 팀 공개홈 기록 탭(행+세그먼트) · 선수 대시보드(스탯 경계) · 공개 선수 프로필(친선 미노출) · B1 입력 화면(헤더·필드 정리)

## Claude Code 첫 프롬프트 예시
```
Handoff/Design.FriendlyMatch/README.md와 두 dc.html을 읽어.
1) Match에 MatchType(Official/Friendly) 추가 + 기존 B1 저장분 Friendly 마이그레이션,
   순위 재계산 Usp가 Official만 집계하도록 수정.
2) B1 입력 화면 정리: 제목·부제 변경, 대회/리그 필드 제거, 저장=Friendly 고정.
3) 결과 목록에 친선 행 스타일(점선+배경+회색 라벨+회색 승패 뱃지)과

4) 시즌 스탯 = 공식만 + "친선경기 별도 +N", 공개 선수 프로필·순위표에 친선 미노출.

```
