## 4. 공통 확인 (모든 시나리오, `it-common.js` 헬퍼로 자동화)

모든 시나리오 스크립트가 페이지 진입마다 자동 수행:

1. 이모지 0 (정규식 스캔) · 오렌지 요소는 허용 목록(랜딩 CTA·카운트 뱃지·화면당 주 버튼 1개)만
2. 한글 줄바꿈 결함 0 — 버튼·pill·메타 `white-space: nowrap` 계산값
3. 모바일 390px: 가로 스크롤 0 · 터치 타깃 44px+ · 입력 font-size 16px(iOS 줌 방지)
4. 폼 오류 토스트 0건(전부 인라인) · 오류 토스트는 자동 소멸 안 함(수동 닫기)
5. 콘솔 에러 0 (수집 후 리포트) · `href="#"` 0건

## 5. 실행 순서·검수 단위

데이터 축적 순서(원본 권장)를 따르되 검수 사이클에 맞춰 3묶음:

1. **1차: S1 + S3** — 게스트·가입 축 + 팀 운영 축 (독립 데이터, 원복 단순) → 보고·검수
2. **2차: S2 + S4** — Claim 축 + 기록 수정 축 (원복 SQL 필수) → 보고·검수
3. **3차: S5 + S6 + S7** — 탐색·경계·빈 상태 (상호 독립, 병렬 가능. S6 에이전트 절은 flag 토글이 있어 맨 마지막) → 보고·검수

각 묶음 종료 시: 검증 데이터 전량 원복 → 신규 스크립트 `Source/Tools/Verification/` 보관 + README 표 갱신 → 이 문서 상단 실행 이력 갱신.

## 헤드리스 주의사항 (요약 — 상세는 Verification/README.md)

- Edge 150+: `puppeteer.launch()` 실패 → `msedge.exe --headless=new --remote-debugging-port` 직접 실행 후 `connect` (`--user-data-dir` 실행마다 고유)
- PC·모바일 트리가 둘 다 렌더 → `getBoundingClientRect().width > 0`으로 보이는 쪽만
- 리다이렉트는 await 뒤 → **URL이 멈출 때까지 대기** 후 판정, 경로 비교 `startsWith` 금지
- 스크린샷 직후 `waitForFunction` rAF 폴링 멈춤 → `{ polling: 300 }`
- 스크립트 파일의 한글: PowerShell 치환 금지(ANSI 파괴) — Edit/Write 도구 또는 bash sed
